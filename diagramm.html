Diagrammid Microsoft Projectis

Diagrammid on visuaalsed tööriistad, mis aitavad projektijuhtidel jälgida ajakava, ressursse ja töömahtu ühe pilguga.

Miks kasutada diagramme?

Microsoft Project pakub erinevaid vaateid (views), sest tekstist ja tabelist ei piisa alati keerulise projekti mõistmiseks. Diagrammid aitavad märgata vigu planeerimises, näha kriitilist teed ja vältida meeskonna ülekoormust.

1. Gantti diagramm (Gantt Chart)

Mis see on: See on kõige populaarsem vaade projektijuhtimises. See näitab ülesandeid horisontaalse graafikuna ajajoonel.

Kuidas avada: Vali ülemisest menüüst View > Gantt Chart.

Milleks kasulik: Gantti diagramm näitab selgelt, millal ülesanded algavad ja lõppevad ning kuidas need on omavahel seotud (nooled ülesannete vahel). See aitab näha projekti üldist kestust.

2. Ressursside diagramm (Resource Graph)

Mis see on: See diagramm näitab visuaalselt ressursside (töötajate või seadmete) koormust konkreetsel ajahetkel.

Kuidas avada: Vali menüüst View > Other Views > Resource Graph.

Milleks kasulik: See on parim tööriist "ülekoormuse" (overallocation) leidmiseks. Kui tulp on punane, tähendab see, et töötajale on määratud rohkem tööd, kui ta suudab teha (nt 12 tundi 8-tunnise tööpäeva jooksul).

3. Kasutuse diagrammid (Task & Resource Usage)

Mis see on: Need on detailsed vaated, mis kombineerivad tabeli ja kalendri. Task Usage näitab, millised ressursid töötavad ülesande kallal, ja Resource Usage näitab, milliseid ülesandeid üks inimene täidab.

Kuidas avada: Vali View > Task Usage või View > Resource Usage.

Milleks kasulik: See vaade on vajalik täpseks planeerimiseks. Siin saab näha ja muuta töötunde iga päeva kohta eraldi. See annab projektijuhile täieliku kontrolli tööaja üle.

Kokkuvõte

Õige diagrammi valik sõltub sinu eesmärgist. Kasuta Gantti graafikut tähtaegade jälgimiseks, Resource Graphi koormuse kontrollimiseks ja Usage vaateid detailseks tundide planeerimiseks. Kombineerides neid vaateid, saad oma projekti edukalt kontrolli all hoida.
